# To-Do List App with Python

This project is a simple command-line To-Do List application.  
The user can add tasks, view them, mark them as completed, and delete them through a text-based menu.

I created this project as my first step in learning Python and as an introduction to the world of IT and software development. I plan to improve it over time as I learn more.

---

## Why I built this
I built this project to practice writing a complete CLI program from start to finish.
My main focus was building a clear menu flow, handling tasks in memory, and keeping the code readable.

---

## What I learned
- How to design a simple CLI menu and keep the user flow clear
- How to manage a list of tasks (add / show / complete / delete) in a clean way
- Where the weak points are (input validation, structure), and what to improve next

---

## What is this project?
The main purpose of this app is to manage a basic list of tasks.  
The user can:

- Create new tasks  
- See all existing tasks  
- Mark tasks as completed  
- Delete tasks  
- Exit the program through a menu  

My goal with this project is not to build a perfect app, but to have a working, simple program that I can use to practice and gradually improve.

---

## How to run

After starting the program, a simple text-based menu is shown.  
The user can choose options by entering the corresponding number.

To run the app locally:

```bash
python todo.py
```

## Known Issues

The app is still under active development, and there are some known limitations and issues:

- Not all invalid user inputs are handled gracefully yet; in some cases the program may behave unexpectedly

- Tasks are currently stored only in memory and are lost when the program exits

- The code can be refactored to be shorter, cleaner, and more modular

I keep these points here on purpose as a reminder of what I want to improve in future versions.
